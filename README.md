Небольшой проект "Панель управления задачами" с Material Design интерфейсом на Vue.js, Vuetify, Vuex и vue-i18n


ДОРАБОТКА ПРОЕКТА

В проект добавлены слоты Vue.js для создания гибких и переиспользуемых компонентов.

Добавленный функционал:

1) Созданы компоненты со слотами:
   - BaseCard - универсальная карточка со слотами header, content, actions
   - BaseDialog - универсальное модальное окно со слотами title, content, actions
   - StatCard - карточка статистики с использованием слотов BaseCard
   - TaskItem - элемент задачи со слотами checkbox, title, subtitle, actions
   - Toolbar - панель инструментов со слотами actions, filters

2) Переработаны страницы с использованием компонентов со слотами:
   - Dashboard.vue теперь использует BaseCard и StatCard
   - Tasks.vue использует BaseCard, BaseDialog, TaskItem, Toolbar

3) Компоненты зарегистрированы глобально в main.js для удобства использования

Структура компонентов со слотами:

src/components/
1) BaseCard.vue - базовая карточка
   - слот header - заголовок карточки
   - слот content - основное содержимое
   - слот actions - кнопки действий
   - слот default - стандартное содержимое

2) BaseDialog.vue - базовое модальное окно
   - слот title - заголовок окна
   - слот content - основное содержимое
   - слот actions - кнопки внизу

3) StatCard.vue - карточка статистики
   - наследует слоты BaseCard
   - слот title - заголовок
   - слот value - значение
   - слот extra - дополнительный элемент

4) TaskItem.vue - элемент задачи
   - слот checkbox - чекбокс с доступом к данным задачи
   - слот title - название задачи
   - слот subtitle - подзаголовок (приоритет, дата)
   - слот actions - кнопки действий

5) Toolbar.vue - панель инструментов
   - слот actions - кнопки справа
   - слот filters - фильтры под панелью

Технологии:

1) Vue.js 3
2) Vuetify 3
3) Vue Router 4
4) Vuex 4
5) vue-i18n 9
6) Vite
7) MDI Icons
8) flag-icons

Особенности реализации со слотами:

1) Все повторяющиеся элементы интерфейса вынесены в переиспользуемые компоненты
2) Слоты позволяют кастомизировать содержимое без изменения компонентов
3) Компоненты BaseCard и BaseDialog служат обертками для единого стиля
4) TaskItem получает данные задачи через props и передает их в слоты
5) Глобальная регистрация компонентов упрощает их использование на страницах
6) Один компонент может использоваться для разных целей благодаря слотам


Команды для запуска:

Установка зависимостей:
npm install

Запуск в режиме разработки:
npm run dev

Сборка для production:
npm run build
